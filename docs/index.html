The first screen is the proxy client, not a marketing page.

ClashMax aims for moderate density, explicit state, and quick operation. Common pages are built around Mihomo's control plane.

01 Dashboard Start, stop, run mode, system proxy, and TUN state.
02 Profiles Import local YAML, manage subscriptions, update, rename, and delete.
03 Proxies Switch proxy groups, test latency, and run provider health checks.
04 Connections Inspect sessions, destinations, matched rules, and live traffic.
05 Rules Review runtime rule lists and trace configuration hit paths.
06 Logs Watch Mihomo logs while debugging config and network issues.
07 Settings Manage the core, helper, update source, and runtime parameters.
08 Menu Bar Check status and run lightweight actions from the menu bar.

Security boundaries are explicit, and privileged paths stay narrow.

A proxy client touches system network settings, so default behavior must be concrete, inspectable, and recoverable.

Local controller

The Mihomo controller listens on 127.0.0.1 by default, so the control plane is not exposed to the LAN.

Per-run secret

Each launch generates a new controller secret and uses Bearer auth for Mihomo REST/WebSocket access.

Helper boundary

Normal system proxy runs through the user-mode core. TUN capabilities go through the privileged helper, with app-owned core/config paths validated.
